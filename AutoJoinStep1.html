Automatic foreign key detection to JOIN tabular data from disparate sources

x
 

As Chris pointed out in this analysis, there is currently no support in the Pandas ecosystem for automatically detecting relations between tables, and this is a painful task.

I've started prototyping a methodology to address these pain points. In this first example, we're using fuzzy matching between columns to automatically identify and recommend joins between tables.

This methodology recommended 4 table JOINs that Chris had previously identified through a painstakingly manual process (SATs-ClassSize, ProgReports-SATs, District-ProgReports, and AttendEnroll-ProgReports). Now that we can do fuzzy joins, it identified the school name as a potentially more reliable field to join on than mangled versions of the school ID for some of these joins. Additionally, it highlighted several other relations between tables that weren't present in Chris's methodology (which was only focused on the ones that would trace back to the dependent variable). These were AttendEnroll-ClassSize and ClassSize-District.

 
At a high level, this methodology looks at each pair of tables, and then each pair of columns between these two tables, and scores these columns according to the likelihood that they represent a relation between the two tables. It then returns recommendations where the score exceeds a certain cutoff. A hybrid automated-manual system would display recommendations the system is less sure about to the user, who can then affirm that this is a foreign key relation or indicate that it is not.
